Figure 7.4 — System architecture. The system is organised into four tiers: the Frontend (vanilla JavaScript SPA with three scoring mode options), the Backend (FastAPI server encapsulating the SemanticEngine, API routing layer, and ClaimFilter), the ML Layer (GPU-resident transformer and serialised probe bundle), and the offline Training Pipeline (Jupyter notebooks producing the probe bundles loaded at runtime). Solid arrows indicate live request/response flow; dashed arrows indicate offline artefact loading.
