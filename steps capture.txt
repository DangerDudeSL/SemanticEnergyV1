The Probes — How They Work

There are 4 trained probes, all logistic regression classifiers:

Probe	Token Position	Layer Range	What it predicts
TBG Energy	Last prompt token (before generation)	28–32	High/low energy (confidence)
TBG Entropy	Last prompt token (before generation)	21–25	High/low entropy (uncertainty)
SLT Energy	Second-to-last generated token	17–21	High/low energy
SLT Entropy	Second-to-last generated token	20–24	High/low entropy
Token positions:

TBG (Token Before Generation) — the last token of the formatted prompt (<|eot_id|> in Llama 3). This is the model's internal state before it has written a single word. It encodes how much the model "knows" about the topic.
SLT (Second-to-Last Token before EOS) — the second-to-last generated token, just before the model decides to stop. This is the model's state after generating the full answer, encoding how confident it was throughout.
Score generation pipeline:


Prompt → forward pass → hidden states (33 layers × 4096 dims)
                              ↓
         Extract layers 21-25 (TBG entropy) → flatten to (1, 4×4096)
                              ↓
         StandardScaler → LogisticRegression.predict_proba()
                              ↓
         entropy_risk = prob[:, 1]   (high entropy = high uncertainty = high risk)
         energy_risk  = 1 - prob[:, 1]  (energy is confidence, invert for risk)
                              ↓
         combined_risk = (energy_risk + entropy_risk) / 2
Why these layers? The sweep across all 33 layers found that mid-to-late layers (14–25) carry the most hallucination signal — this matches the literature where factual knowledge is encoded in middle transformer layers.

Per-sentence hallucination — Yes, extendable. The framework already has the building block in 04_sentence_baseline.ipynb. The extension would be:

Generate the answer (already done in SLT mode)
Split into sentences using pysbd (already in notebook 04)
Run a separate forward pass on prompt + answer[:sentence_end] for each sentence boundary
Extract SLT hidden states at that boundary position
Run the SLT probe — each sentence gets its own risk score
Return the highest-risk sentence highlighted
This is more expensive (N forward passes for N sentences) but much cheaper than the Full SE multi-generation approach.

I found three bugs in the JS and one caching issue causing the layout problem:

Layout on refresh — browser caches styles.css/script.js. New HTML (mode selector markup) loads but old CSS/JS serve from cache — buttons appear unstyled and JS event handlers are stale.
Output not displaying — Full SE mode registers two setTimeout calls for status text that are never cancelled. They fire at 7s and 14s even after the response arrives, overwriting "Ready" and confusing the UI state.
Error color never resets — statusText.style.color is set red on error but never cleared, so it stays red on the next request.
No null guard on data.answer — if the backend returns an error JSON with HTTP 200, addMessage(undefined) throws silently.
